Security Notes.

These notes explain how protected areas are intended to be accessed and how security concerns can be reported.

Authorised access

Private pages and worker API routes are intended for authorised users only. Login, session, and server-side verification checks protect those areas.

Session safety

  • Keep login details private.
  • Sign out when using a shared device.
  • Clear browser cookies if a device is lost or no longer trusted.
  • Report unexpected login or session behaviour.

Reporting concerns

If you believe you have found a security issue, contact mason@mas0ng.com with enough detail to understand and reproduce the issue.
