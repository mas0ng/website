Security notes

Keeping private apps protected.

These notes explain intended access controls, session checks, server-side secrets, upload limits, and how to report security concerns.

Last updated: 1 June 2026

Authorised access

Private pages and Worker API routes are intended for authorised users only. Browser sign-in is handled at auth.mas0ng.com. Session cookies, server-side auth verification, and route checks protect private areas.

AI token protection

AI provider keys should stay server-side in Worker secrets. The browser should call the private AI Worker, and the Worker should verify the session before sending requests to providers such as OpenAI or DeepSeek.

Upload limits

The AI app enforces compressed file limits, total upload allowance, same-origin checks for unsafe requests, and retention rules for pinned and recent chats. These controls reduce storage abuse and accidental cost exposure.

Session safety

  • Keep login details private.
  • Sign out when using a shared device.
  • Clear browser cookies if a device is lost or no longer trusted.
  • Report unexpected login, session, AI usage, upload, or dashboard behaviour.

Reporting concerns

If you believe you have found a security issue, contact mason@mas0ng.com with enough detail to understand and reproduce the issue.
